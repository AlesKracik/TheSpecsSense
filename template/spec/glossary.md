# Glossary

> Shared vocabulary for the spec. One canonical term per concept; ban synonyms.
> Agents propose additions via PR; the domain expert curates merges.
> When a term is added here, it must also appear in `round-1/entities.json`,
> `round-1/verbs.json`, or `round-1/actors.json` as a structured record.

## Conventions

- **Bold** for the canonical term.
- *Italics* on first use of a term being defined within another definition.
- All-caps for state names (e.g. `VERIFYING`, `AWAITING_REVIEW`).
- Snake_case for event names (e.g. `verify_complete_mismatch`).
